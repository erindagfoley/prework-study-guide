HTML

Tags

  • !DOCTYPE html: - Tells the browser that the document is written in HTML
  • html: indicates the start of an HTML document
  • head: - contains the metadata for our web application and is NOT visible to the user
    • meta: defines metadata such as the character set, description, keywords, author, etc. All of these attributes are included in the INSIDE of the element
  • title: defines the title of the website as shown as a tab in the browser
  • header (as opposed to head): appears at the top of the page and usually contains a navigation menu and/or introductory content such as the h1 tag. It includes what WILL BE visible to the user
  • style: defines the CSS styles associated with a website
  • base: defines the base URL for a webpage
  • link: connects an external resource (not contained in the website) to the HTML document
  • script: embeds client-side scripts in the HTML document
  • div: defines a division or a section in an HTML document
  • span: creates an inline division in an HTML document, which does not start on a new line and only takes up as much width as necessary
  • footer: appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation
  • main: Specifies the main content of a document
  • section: creates a section or component in the HTML document to group related content together
  • !-- whatever you want to add -- (within brackets): adds comments to your code that are only visible to developers

Attributes

  • charset: allows us to use most characters and symbols on the page
  • http-equiv: allows us to load web pages using hypertext links
  • name: sets the default view for our webpage
  • class: allows you to style multiple HTML elements the same way
  • id: similar to the class attribute; however it must only be used by one HTML element within the page

Git

  • cd name-of-directory: changes the directory/repo that we are working in
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds files to staging. This is done prior to committing the code to the cloud. -A indicates that we are adding ALL files
  • git commit -m "description": commits the files to the cloud
  • git pull origin branch: pulls the most recent version of code that is in the cloud. We often pull from the main branch, which would look like git pull origin main
  • git push origin branch: pushes the changes that you made locally to the cloud. For example, git push origin feature/name would push a specific branch to the cloud.
