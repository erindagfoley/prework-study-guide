HTML



Tags



  • !DOCTYPE html: - Tells the browser that the document is written in HTML
  • html: indicates the start of an HTML document
  • head: - contains the metadata for our web application and is NOT visible to the user
    • meta: defines metadata such as the character set, description, keywords, author, etc. All of these attributes are included in the INSIDE of the element
  • title: defines the title of the website as shown as a tab in the browser
  • header (as opposed to head): appears at the top of the page and usually contains a navigation menu and/or introductory content such as the h1 tag. It includes what WILL BE visible to the user
  • style: defines the CSS styles associated with a website
  • base: defines the base URL for a webpage
  • link: connects an external resource (not contained in the website) to the HTML document
  • script: embeds client-side scripts in the HTML document
  • div: defines a division or a section in an HTML document
  • span: creates an inline division in an HTML document, which does not start on a new line and only takes up as much width as necessary
  • footer: appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation
  • main: Specifies the main content of a document
  • section: creates a section or component in the HTML document to group related content together
  • !-- whatever you want to add -- (within brackets): adds comments to your code that are only visible to developers


Attributes



  • charset: allows us to use most characters and symbols on the page
  • http-equiv: allows us to load web pages using hypertext links
  • name: sets the default view for our webpage
  • class: allows you to style multiple HTML elements the same way
  • id: similar to the class attribute; however it must only be used by one HTML element within the page


CSS



Vocabulary



  • Inline CSS - uses a style attribute to style a particular page element only
  • Internal style sheet - an embedded stylesheet directly in the HTML file using a style element in the head element. The rules only apply to the page
  • External style sheet - When you have a separate CSS file that is linked within the HTML
  • Separation of concerns - a widely used term in web development and refers to separating code into different files based on use.
  • Selector, property, value, declaration, and declaration block - The parts of a CSS style rule
  • Padding selector - Makes an element larger
  • Do Not Repeat Yourself (DRY) - a coding principle that encourages developers to reduce the number of lines of code


Selectors



  • * selector - Used to format ALL elements on a page
  • Other selectors include h1-h6 (headings), img (images), etc. They are the same as the elements in the HTML document


Properties



  • Margin property - Creates whitespace along the entire edge of a page
  • Padding property - Increases the space around an element to make the entire element larger
  • How to apply a property/value pair to two elements at once - Add commas to separate the two elements
  • Block value - assigns each img to start on a new line

Git



Vocabulary



  • Repo - a folder for your project that contains all of the files and their revision history
  • Branch - a file that exists in the repo, like how you would have different files in a folder
  • Commit - to save your changes to a local repo on your device
  • Pull - To take information from a cloud repo, such as GitHub, and add it to your local repo
  • Push - To take information from a local repo and add it to a cloud repo, such as GitHub
  • Pull request (PR) - A request to others who are working in your repo to approve and add the changes that you made
  • Merged/pulled in - the publishing of changes from a pull request into the live site
  • Issue - a way to report a new task or report content fixes


Commands



  • cd name-of-directory: changes the directory/repo that we are working in
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds files to staging. This is done prior to committing the code to the cloud. -A indicates that we are adding ALL files
  • git commit -m "description": commits the files to the cloud
  • git pull origin branch: pulls the most recent version of code that is in the cloud. We often pull from the main branch, which would look like git pull origin main
  • git push origin branch: pushes the changes that you made locally to the cloud. For example, git push origin feature/name would push a specific branch to the cloud.

JavaScript



Vocabulary



  • Variable - a named container that allows us to store data in our code.
  • Data Types - values that are stored in a variable. They include strings (text), numbers, booleans (true/false statements), arrays (groups of related data), and objects (a mutable collection of key-value pairs)
  • Control flow - the order in which a computer executes code in a script.
  • Key - describes a value. For example, name: "Erin"; name is the key
  • Value - the data that corresponds with a key. For example, name: "Erin"; Erin is the value
  • Operators - mathematical symbols that produce a result based on two values
  • Conditionals - tests if an expression is true or false
  • Array - a single variable that is used to hold a group of data
  • For loop - used to log multiple topics from an array in the console
  • Control flow - the order in which a computer executes code in a script
  • Method - actions that objects can perform
  • Arguments - values passed as input to a function that go after the method in parentheses
  • Functions - a way of including multiple commands in JS using one name (or functionality) that you wish to reuse over and over
  • Arguments - values passed as input to a function
  • Variable scoping - The concept that if a variable is defined inside a function, that variable is only available INSIDE the function
  • Event handler - code structures that listen for activity in the browser, and run code in response
  • Weakly typed language - a programming language in which you don't need to specify the data type when declaring a variable. It also using type coercion, which is explained below
  • Type coercion - rules that make it so that if a developer uses the incorrect data type, it will convert the value to the correct data type that is expected
  • Primitives - data types that are immutable, meaning that their values cannot be changed without creating an entirely new value in your code
  • Objects - a collection of key-value pairs. They are the only mutable data type (a data type whose value CAN be changed without creating an entirely new value in your code)
  • JSON (JavaScript Object Notation) - a text-based format that is independent of any programming language


Commands



  • const - used to create a new variable in which the the variable should not change
  • let - used to create a new variable whose value can change
  • + - * / operators - used to add, subtract, multiply, and divide
  • = (a.k.a. assignment operator) - assigns a value to a variable
  • === (a.k.a. strict equality) - performs a test to determine if two values are equal and of the same data type. It returns a true/false result
  • !== (a.k.a. does-not-equal) - returns the OPPOSITE result of the strict equality operator
  • if…else - sends back one value if a condition is true and will send a different value if a condition is false
